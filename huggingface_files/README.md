## Technical Details

This tokenizer uses the Byte Pair Encoding (BPE) algorithm:

1. **Character Initialization**: Starts with all unique characters
2. **Pair Frequency Analysis**: Counts adjacent character/token pairs
3. **Iterative Merging**: Merges most frequent pairs into new tokens
4. **Vocabulary Building**: Continues until target vocabulary size is reached

### Why BPE for Odia?

- **Handles Complex Script**: Odia uses a syllabic alphabet with many ligatures
- **Efficient Encoding**: Achieves high compression ratios
- **Handles Rare Words**: Can break unknown words into known subwords
- **Language Agnostic**: Works well across different writing systems
